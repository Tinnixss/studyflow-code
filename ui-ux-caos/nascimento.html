🎓 O que você está praticando (e sofrendo)
Anti-Padrões de UI/UX O que torna uma interface terrível? Estudo prático criando o oposto do ideal para entender acessibilidade e UX.
Manipulação Avançada do DOM Uso intensivo de HTML e JS para criar comportamentos não convencionais — botões que fogem, sliders imprecisos, inputs sabotadores.
Gestão de Eventos Aprofundamento em Event Listeners: mouseover, click, input, drag para controlar o caos criado.
Persistência com IndexedDB Capturar o dado final e salvá-lo de forma estruturada e assíncrona no banco de dados local do navegador.
🎯 Objetivos de Aprendizagem
  • Exercitar a lógica de programação e a manipulação avançada de elementos HTML e CSS através do JavaScript.
  • Compreender na prática o valor de uma boa Experiência do Usuário (UX), sentindo a frustração de operar uma interface mal desenhada.
  • Consolidar o uso do IndexedDB, garantindo que mesmo os dados gerados pelas interfaces mais caóticas sejam validados e salvos corretamente no banco local.
  • Garantir a Ética no Desenvolvimento: aprender a ter liberdade criativa total dentro de limites profissionais (proibição estrita de conteúdos ilegais, ofensivos ou de cunho sexual).
📉 Os Anti-Padrões Implementados
👆
Passo 1 — O Inferno dos Cliques Esforço excessivo para tarefa simples. Cada clique = 1 dia. Precisa de até 31 cliques.
🏃
Passo 2 — A Dança da Frustração Slider com valor oculto durante o arraste. Botão que foge do cursor ao ser aproximado.
🎲
Passo 3 — A Roleta dos Dígitos 60% de chance de digitar um número adjacente ao clicado. Precisão é opcional.
🚀 Atividades Práticas
  • Brainstorming do Caos: planejem qual será a "pior maneira de inserir uma data de nascimento". Regra de ouro: criatividade livre, desde que o conteúdo seja 100% apropriado. Exemplos: botão que precisa de 2.000 cliques para o ano 2000, slider com números invisíveis, teclado que digita aleatoriamente.
  • Construção da Interface (HTML/CSS/JS): no index.html e style.css, construam a interface. No controller.js, programem os eventos usando JS. Use a IA para criar funções mirabolantes — ex: "como faço um botão mudar de posição toda vez que o mouse passar por cima?"
  • A Recompensa — Salvando no IndexedDB: após o usuário "sofrer" para inserir a data, criem um botão "Salvar Data" que captura o valor final, monta o objeto e chama a função de adicionar ao banco do db.js.
  • Mão na Massa e Versionamento: testem abrindo o index.html diretamente no navegador — sem Live Server, para evitar erros de CORS. Mantenham o Console (F12) e a aba Application > IndexedDB abertos para verificar se os dados foram salvos. Quando funcionar, façam o commit e subam para o GitHub.
⚠️ Ética no Desenvolvimento: A liberdade criativa aqui é total, mas dentro de limites profissionais e éticos. É estritamente proibido criar conteúdos ilegais, ofensivos, preconceituosos ou de cunho sexual — mesmo que "apenas para fins didáticos". O caos é bem-vindo; o desrespeito, nunca.
